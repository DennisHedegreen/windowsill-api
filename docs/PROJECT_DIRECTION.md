# Windowsill Project Direction

Windowsill is an API-first open plant data and growing recommendation project.

It combines:

- location
- climate
- growing context
- plant data
- AI-assisted contribution workflows
- human responsibility
- source traceability
- visible uncertainty
- future taxonomic review
- future community field signal

The API remains the center. Websites, widgets, apps, games, hardware and Hedegreen Research TID surfaces may use the API, but they are consumers of the system, not the core identity.

## One-Sentence Direction

Windowsill should become an API-first, AI-assisted, human-accountable open plant knowledge system with visible uncertainty, expert disagreement and community field signals.

## Core Guardrails

1. The API is the center.
2. AI assists but does not authoritatively decide.
3. Sources must remain visible.
4. Uncertainty must remain visible.
5. Expert review and community signal must be separate.
6. Disagreement should be shown, not hidden.
7. Phone-based contribution should be possible.
8. Do not build around one app.
9. Do not fake academic approval.
10. Do not make plant count more important than data quality.

## Working Method

```text
object
-> sources
-> AI-assisted draft
-> human responsibility
-> uncertainty notes
-> review
-> disagreement
-> public entry
-> correction over time
```

For Windowsill, the object is a plant entry.

The same method may later apply to other public knowledge objects: claims, sources, tools, places, observations and datasets.

## What Windowsill Is Not

Windowsill is not:

- a chatbot pretending to know botany
- a plant count race
- official academic certification
- a closed recommendation engine
- one app
- AI-generated plant truth

The correct attitude is:

Use AI to go faster, but use sources, humans, review and uncertainty to avoid lying.
