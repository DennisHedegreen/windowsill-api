# Contribute From Phone

Status: design direction, partly possible through GitHub issues or pull requests.

Windowsill should be possible to contribute to from a phone.

People with real local plant knowledge may not have a full developer setup. The contribution flow should respect that.

## Simple Phone Flow

1. Find a plant idea.
2. Gather a few links or notes.
3. Ask an AI tool to prepare a candidate contribution work pack.
4. Read and correct the output.
5. Open GitHub on the phone.
6. Create an issue, discussion or pull request.
7. Include sources and uncertainty.
8. Submit for review.

## Simple Contribution Fields

An ordinary user should be able to suggest:

- plant name
- local name
- country or region
- why the plant matters
- whether they have grown it
- where they grew it
- links or sources
- uncertainty

They should not need to write JSON.

## Full Contribution Path

People who want to prepare a real plant entry should use:

- `plant.json`
- `source_registry.md`
- `field_rationale.md`
- `uncertainty_notes.md`
- `pr_description.md`

See:

- `docs/AI_PLANT_WORKFLOW.md`
- `docs/templates/plant-suggestion.md`
